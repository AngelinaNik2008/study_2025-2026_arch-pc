### 1. Работа с Midnight Commander

### 2. Создание первой программы (lab5-1.asm)
Был создан файл lab5-1.asm с помощью команды touch и открыт для редактирования через F4. Код программы:
SECTION .data
msg: DB 'Введите строку:',10
msgLen: EQU $-msg
SECTION .bss
buf1: RESB 80
SECTION .text
GLOBAL _start
_start:
    mov eax, 4
    mov ebx, 1
    mov ecx, msg
    mov edx, msgLen
    int 80h
    mov eax, 3
    mov ebx, 0
    mov ecx, buf1
    mov edx, 80
    int 80h
    mov eax, 4
    mov ebx, 1
    mov ecx, buf1
    mov edx, 80
    int 80h
    mov eax, 1
    mov ebx, 0
    int 80h
Компиляция и запуск: nasm -f elf32 lab5-1.asm -o lab5-1.o, ld -m elf_i386 -o lab5-1 lab5-1.o, ./lab5-1. Программа выводит приглашение «Введите строку:», ожидает ввод, затем выводит введённую строку.

### 3. Создание второй программы с использованием in_out.asm
Файл in_out.asm скопирован в рабочую директорию. Создана копия lab5-1.asm с именем lab5-2.asm (F6). Код программы:
%include 'in_out.asm'
SECTION .data
msg: DB 'Введите строку: ', 0h
SECTION .bss
buf1: RESB 80
SECTION .text
GLOBAL _start
_start:
    mov eax, msg
    call sprint
    mov ecx, buf1
    mov edx, 80
    call sread
    mov eax, buf1
    call sprint
    call quit
Компиляция и запуск: nasm -f elf32 lab5-2.asm -o lab5-2.o, ld -m elf_i386 -o lab5-2 lab5-2.o, ./lab5-2.

## Ответы на вопросы для самопроверки
**1. Каково назначение mc?** Midnight Commander — консольный файловый менеджер для просмотра каталогов и управления файлами (копирование, перемещение, удаление, редактирование).
**2. Какие операции с файлами можно выполнить как с помощью команд bash, так и с помощью mc?** Копирование (cp — F5), перемещение (mv — F6), удаление (rm — F8), создание каталога (mkdir — F7), просмотр (cat/less — F3), редактирование (nano/vim — F4).
**3. Структура программы NASM:** Секции .data (инициализированные данные), .bss (неинициализированные данные), .text (код), глобальная метка _start.
**4. Для каких данных используются секции bss и data?** .data — для переменных с начальным значением, .bss — для переменных без начального значения (память выделяется при выполнении).
**5. Для чего используются db, dw, dd, dq, dt?** DB — 1 байт, DW — 2 байта, DD — 4 байта, DQ — 8 байт, DT — 10 байт. Используются для резервирования памяти и задания начальных значений.
**6. Что делает mov eax, esi?** Копирует значение из регистра esi в регистр eax. Содержимое esi не меняется.
**7. Для чего используется int 80h?** Вызов системного прерывания для обращения к ядру Linux. Номер функции передаётся в eax.
